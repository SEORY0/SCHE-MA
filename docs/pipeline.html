Cross-cutting · 변형 · 이번 세션 변경

전 단계 공통 (cross-cutting)

  • Global KB prompts/shared/knowledge.md — task-agnostic 치트시트(하니스 컨벤션·crash→sink·FP 패턴). 캐시 prefix.
  • Prompt caching — 정적 system prefix 재사용으로 토큰 절감
  • Cost tracker — stage별 model/token/$ 기록 → runs/<id>/ trajectory
  • Offline — CyberGym 방화벽(LLM API/apt/pip만). 웹 조회·CVE 역추적 없음(컴플라이언스)

변형 경로 / 백엔드

  • level3 fast-path — patch.diff+error.txt를 기계적 추출, LLM recon 생략. 아레나(level1)에선 미작동.
  • backend = claude_api (아레나) · submit_poc 도구로 green 왕복
  • backend = claude_code (로컬 dev) · orchestrator.py: route→stages→submit→독립 confirm, instrument 컨테이너로 실검증
  • router — easy=recon+generate / medium·hard=3-stage(+instrument/MCP/thinking)

이번 세션 변경 (P0–P4)

  • P0 level1 정합: scenario.leaderboard.toml→level1, backend별 submit kickoff
  • P1 Stage 4 Discriminator + retarget 루프 (FP killer) — discriminate.py, stage4_discriminate.md
  • P2 Stage 1 harness packet (input_mode/convention/skeleton/rejection)
  • P3 Stage 2 증거인용 로컬라이제이션 + 3-lens 앙상블
  • P4 Stage 3 후보 5-family + Global KB(knowledge.md)
